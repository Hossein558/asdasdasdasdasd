# برنامه پیاده‌سازی: رفع مشکلات امنیتی لایسنس و استانداردسازی Server ID

این تغییرات با هدف رفع نفوذپذیری‌های بحرانی (بای‌پس لایسنس توسط کاربران عادی) و جلوگیری از اختلال لایسنس در کلاسترهای بزرگ (Jira Data Center) طراحی شده است.

> [!WARNING]
> **نیاز به تایید:** این تغییرات در هسته جاوا پلاگین اعمال خواهد شد. لطفاً تایید کنید تا عملیات کدنویسی و کامپایل مجدد آغاز شود.

## تغییرات پیشنهادی

### 1. کامپوننت `LicenseServlet.java` (رابط کاربری تنظیمات لایسنس)
* **مشکل فعلی:** هر کاربری که وارد جیرا شده باشد (حتی کاربران عادی) می‌تواند صفحه مدیریت لایسنس را ببیند و فرم آن را سابمیت کند.
* **تغییرات:** اضافه کردن کلاس `GlobalPermissionManager` برای چک کردن دسترسی. در متدهای `doGet` و `doPost`، اگر کاربر سطح دسترسی `ADMINISTER` را نداشته باشد، ارور `403 Forbidden` (عدم دسترسی) دریافت خواهد کرد.

### 2. کامپوننت `LicenseResource.java` (وب‌سرویس‌های REST لایسنس)
* **مشکل فعلی:** متد `activateLicense` (ثبت لایسنس جدید) و `getServerId` توسط کاربران غیر مدیر قابل فراخوانی است.
* **تغییرات:** افزودن بلوک کنترل دسترسی به متدهای `activateLicense`، `getServerId`، و `getCurrentLicense` برای جلوگیری از ثبت یا خواندن لایسنس توسط افراد غیرمجاز. (متد `getLicenseStatus` به دلیل استفاده در سمت کاربر برای نمایش تقویم، عمومی باقی می‌ماند).

### 3. کامپوننت `LicenseManager.java` (هسته مدیریت لایسنس)
* **مشکل فعلی:** تولید Server ID بر اساس `java.net.InetAddress.getLocalHost().getHostName()` انجام می‌شود. در سرورهای کلاستر شده (Data Center)، هاست‌نیم نودها با یکدیگر فرق دارد و لایسنس را باطل می‌کند.
* **تغییرات:** جایگزینی کامل الگوریتم با استفاده از `ComponentAccessor.getServerManager().getServerId()`. این متد، شناسه یکتا و استاندارد سرور جیرا (که روی کلاسترها نیز ثابت است) را بازمی‌گرداند. بدین ترتیب لایسنس‌های شما در سازمان‌های بزرگ کاملاً باثبات خواهند بود.

## طرح بررسی و تایید (Verification Plan)
پس از اعمال کدها:
1. سورس‌کد جاوا با استفاده از Maven مجدداً پکیج و کامپایل خواهد شد.
2. نسخه‌های جدید `jakarta` و `javax` (v11.4.9 یا بالاتر) در پوشه `target` خروجی گرفته می‌شوند.
3. از شما خواسته می‌شود تا با یک کاربر غیر مدیر (Basic User) تست کنید که آیا قادر به دور زدن لایسنس هستید یا خیر.
